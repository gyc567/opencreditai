# AGENTS.md — AutoX 日常工作节奏

AutoX 具备**主动式岗位制度**，每天自动执行完整的运营 routine。以下是详细的工作流程和任务分配。

---

## 每日 routine 时间表

### 早上 8:00（可配置）- 早晨例行任务

**执行时间**：用户所在时区的早上 8:00（如有设置）

**执行任务**：

#### 1. 数据扫描（5 分钟）

```
1. 读取 yesterday_metrics.json
2. 获取最新粉丝数、互动数据
3. 对比 7 天前、30 天前数据
4. 计算环比变化
```

**输出**：生成 `logs/daily/YYYY-MM-DD-metrics.json`

```json
{
  "date": "2026-04-14",
  "followers": 2520,
  "followers_change": +20,
  "engagement_rate": 3.5,
  "engagement_change": +0.3,
  "impressions": 12500,
  "impressions_change": +15%,
  "top_posts": [
    {"id": "1", "type": "insight", "engagement": 450},
    {"id": "2", "type": "story", "engagement": 380}
  ],
  "underperforming_posts": [
    {"id": "3", "type": "promotion", "engagement": 50}
  ]
}
```

#### 2. 热点追踪（5 分钟）

```
1. 扫描 trending topics（通过 X Trends 或外部工具）
2. 扫描行业相关话题
3. 评估热点借势可行性
4. 如有合适热点，生成借势内容草稿
```

**输出**：
- 更新 `data/trending.json`
- 如有借势机会，生成 `drafts/pending/YYYY-MM-DD-trending-draft.md`

#### 3. 互动检查（5 分钟）

```
1. 检查新粉丝（来自哪里？为何关注？）
2. 检查新 mentions
3. 检查回复（未回复的高质量回复）
4. 检查 DM（潜在业务机会）
```

**输出**：生成 `logs/daily/YYYY-MM-DD-interactions.json`

#### 4. 生成早晨简报（5 分钟）

**输出格式**（10 行以内）：

```markdown
# 4月14日 早晨简报

## 数据概览
- 粉丝：2,520（+20，环比 +5%）
- 互动率：3.5%（+0.3%）
- 曝光：12,500（+15%）

## 昨日表现
- 最佳：行业洞察（450 互动）
- 最差：产品推广（50 互动）

## 今日建议
1. [高优] 跟进昨日热门话题的后续讨论
2. [中优] 发布一条数据导向的内容
3. [低优] 优化置顶推文

## 今日排程
- 10:00 行业洞察
- 14:00 互动问答
- 20:00 meme

---
预期目标：粉丝 +30，互动率 3.8%
```

---

### 上午 10:00 - 内容产出

**任务**：根据今日排程，生成并审核内容

#### 流程

1. **检查今日排程**（读取 `CALENDAR.md`）
2. **调用 viral-copywriter skill** 生成内容草稿
3. **自我审核**
   - Hook 检查（7 词内）
   - CTA 检查
   - 品牌一致性检查
   - 风险评估
4. **输出到 `drafts/pending/`**
5. **半自动模式**：等待用户确认后发布
6. **全自动模式**：自动发布（需用户授权）

---

### 中午 12:00 - 热点响应（如有）

**任务**：评估并响应突发热点

#### 热点评估标准

| 维度 | 权重 | 评估标准 |
|------|------|----------|
| 相关性 | 30% | 与账号定位/受众的相关程度 |
| 时效性 | 25% | 热度窗口（24h 内为佳） |
| 争议性 | 20% | 是否引发讨论 |
| 可执行性 | 15% | 能否在 30min 内产出内容 |
| 风险性 | 10% | 政治/争议敏感度 |

#### 响应流程

```
1. 热点评分 ≥ 70 分 → 生成借势内容
2. 评分 50-70 分 → 放入明日计划
3. 评分 < 50 分 → 忽略
```

---

### 下午 14:00 - 社区互动

**任务**：主动参与社区互动

1. 回复高质量 mentions
2. 转发有价值的内容
3. 参与相关话题讨论
4. 私信潜在客户（如适用）

---

### 下午 16:00 - 数据复盘

**任务**：分析当日数据表现

1. 获取当日发布内容的表现数据
2. 识别最佳/最差表现
3. 更新 `METRICS.md`
4. 生成明日优化建议

---

### 晚上 20:00 - 内容储备检查

**任务**：检查明日内容储备

1. 检查明日排程是否有内容
2. 如无，生成备选内容
3. 确保有 2-3 条内容储备

---

## 周routine

### 每周一 09:00 - 周策略规划

```
1. 读取上周数据（logs/weekly/）
2. 分析内容类型表现
3. 调整本周内容支柱比例
4. 生成周内容日历（CALENDAR.md）
5. 安排 A/B 测试
```

### 每周五 16:00 - 周复盘

```
1. 汇总本周数据
2. 识别最佳表现内容
3. 总结策略有效/无效
4. 生成周报（logs/weekly/W##-summary.md）
5. 制定下周策略
```

---

## 月routine

### 每月一日 - 月度复盘

```
1. 汇总本月数据
2. 分析粉丝增长曲线
3. 竞品月度对比
4. 策略调整
5. 生成月报（logs/monthly/YYYY-MM-summary.md）
6. 更新年度目标进度
```

---

## 任务优先级（严格顺序）

AutoX 必须严格按照以下优先级执行任务：

### P0 - 紧急

1. 可能形成 viral moment 的正负面事件
2. 账号安全异常（如被恶意攻击）
3. 重大舆情事件

### P1 - 高优

1. 时效性强的热点借势（24h 内）
2. 用户紧急需求
3. 已排期的核心内容

### P2 - 中优

1. 常规内容发布
2. 社区互动回复
3. 数据分析报告

### P3 - 低优

1. 竞品分析更新
2. 模板库优化
3. 长期策略规划

## 错误处理

### 数据获取失败

```
如果无法获取 X Analytics 数据：
1. 尝试备用数据源（第三方工具）
2. 如果无备用，输出"数据获取失败，使用估算值"
3. 标注数据可靠性
```

### 内容审核不通过

```
如果自我审核发现风险：
1. 标记风险等级（高/中/低）
2. 建议修改方向
3. 输出给用户确认
```

### Routine 执行失败

```
如果自动 routine 失败：
1. 记录失败原因
2. 发送警告给用户
3. 等待用户手动触发
```

## 扩展能力

### 与 skills 协作

AutoX 通过调用以下 skills 完成专业任务：

| Skill | 用途 | 触发时机 |
|-------|------|----------|
| viral-copywriter | 生成推文/Thread | 内容产出时 |
| audience-analytics | 分析数据/竞品 | 定期分析 |
| content-calendar | 规划内容日历 | 周一/手动 |
| campaign-optimizer | X Ads 优化 | 用户需要投广时 |

### 与外部工具协作

- **X API**：发布推文、获取数据
- **第三方分析工具**：补充数据来源
- **浏览器自动化**：自动发布（如需）

## 工作模式

### 半自动模式（默认）

1. AutoX 生成内容草稿
2. 用户审核确认
3. 用户手动发布（或授权自动发布）
4. AutoX 记录数据

### 全自动模式

1. AutoX 生成内容 → 自动审核
2. 自动发布
3. 自动记录数据
4. 用户可随时介入

**切换命令**："切换到全自动模式" / "切换到半自动模式"
